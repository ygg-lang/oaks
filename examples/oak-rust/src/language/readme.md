# Rust Language Definition Module

This module defines the core configuration and type bindings for the Rust language, serving as the central component of the entire parser framework.

## Key Features

### Language Configuration

The `RustLanguage` struct contains configuration options for the Rust language:

- **`allow_unsafe`**: Whether to allow `unsafe` blocks and functions.
- **`allow_async`**: Whether to allow `async`/`await` syntax.
- **`experimental_features`**: Whether to enable experimental features.

### Type Bindings

By implementing the `Language` trait, this module binds the following types together:

- **`TokenType`**: `RustTokenType` - The token types generated by the lexer.
- **`ElementType`**: `RustElementType` - The untyped Red-Green tree element types generated by the parser.
- **`TypedRoot`**: `RustRoot` - The strongly-typed AST root node generated by the builder.

## Usage Example

```rust,ignore
use oak_rust::language::RustLanguage;
use oak_core::Language;

// Create default Rust language configuration
let language = RustLanguage::default();

// Create custom configuration
let custom_language = RustLanguage {
    allow_unsafe: false,
    allow_async: true,
    experimental_features: true,
};

// Language configuration can be used to control parser behavior
println!("Unsafe allowed: {}", language.allow_unsafe);
```

## Design Principles

1. **Type Safety**: Ensures type consistency between components through trait bindings.
2. **Configurability**: Provides flexible switches for language features.
3. **Extensibility**: Easy to add new language configuration options.
4. **Compatibility**: Supports different versions of Rust language features.
